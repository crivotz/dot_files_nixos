# NixOS Configuration

## Quick reference

```bash
# Apply .nix changes (without updating packages)
sudo nixos-rebuild switch --flake . --impure

# Update all packages (flake inputs) and rebuild
nix flake update && sudo nixos-rebuild switch --flake . --impure

# Preview changes without applying
sudo nixos-rebuild dry-activate --flake . --impure

# Rollback to previous generation
sudo nixos-rebuild switch --rollback

# Clean old generations and free disk space
sudo nix-collect-garbage -d

# List available generations
sudo nix-env --list-generations --profile /nix/var/nix/profiles/system
```

## First install

### 1. Install NixOS

Download the ISO from https://nixos.org/download and install choosing **no desktop environment** (minimal).

### 2. Enable flakes (only during pre-rebuild period)

```bash
mkdir -p ~/.config/nix
echo "experimental-features = nix-command flakes" >> ~/.config/nix/nix.conf
```

### 3. Clone this repo

```bash
nix-shell -p git
git clone git@github.com:crivotz/dot_files_nixos ~/dot_files_nixos
```

`nubem_dot_files` and `nv-ide` are cloned automatically by `home.activation` on the first `nixos-rebuild switch` and kept up to date with pull on subsequent runs.

### 4. Link hardware-configuration.nix

```bash
# desktop
cp /etc/nixos/hardware-configuration.nix ~/dot_files_nixos/hosts/NIXMAU/hardware-configuration.nix
# laptop
cp /etc/nixos/hardware-configuration.nix ~/dot_files_nixos/hosts/NIXMAU_LT/hardware-configuration.nix
```

### 5. Apply the configuration

```bash
sudo nixos-rebuild switch --flake ~/dot_files_nixos --impure
```

### 6. Post-rebuild setup (once only)

```bash
# Neovim: disable Mason on NixOS
cat > ~/.nv-ide/lua/plugins/nix-mason-override.lua << 'EOF'
return {
  { "williamboman/mason.nvim",                    enabled = false },
  { "williamboman/mason-lspconfig.nvim",          enabled = false },
  { "WhoIsSethDaniel/mason-tool-installer.nvim",  enabled = false },
}
EOF
echo "lua/plugins/nix-mason-override.lua" >> ~/.nv-ide/.gitignore
```

### 7. Reboot

```bash
sudo reboot
# dms-greeter appears → select Sway or Hyprland → log in
```

## System updates

### Apply .nix file changes (without updating packages)

```bash
cd ~/dot_files_nixos
sudo nixos-rebuild switch --flake . --impure
```

Uses package versions already pinned in `flake.lock`. Fast, downloads little or nothing.

### Update all packages (like apt upgrade)

```bash
cd ~/dot_files_nixos
nix flake update                              # updates flake.lock to latest versions
sudo nixos-rebuild switch --flake . --impure
```

Updates nixpkgs, home-manager, neovim-nightly, dms, etc. May download updates.

### Remove software and free space

Remove the package from the `.nix` configuration, then:

```bash
sudo nixos-rebuild switch --flake . --impure  # removes from active profile
sudo nix-collect-garbage -d                   # deletes old generations and frees space
```

Removed packages stay in `/nix/store` to allow rollback. The garbage collector deletes them permanently.

### Rollback (revert to previous version)

```bash
sudo nixos-rebuild switch --rollback
```

If the system doesn't boot, select the previous generation from the boot loader menu (systemd-boot) at startup.

### List available generations

```bash
sudo nix-env --list-generations --profile /nix/var/nix/profiles/system
```

### Delete old generations

```bash
sudo nix-collect-garbage -d                        # delete all generations except the active one
sudo nix-collect-garbage --delete-older-than 7d    # keep only the last 7 days
```

After cleanup, rollback to deleted generations is no longer possible.

## Ruby on Rails multi-version with devenv

**devenv**: each project declares its own Ruby and Node versions.

```bash
cd ~/Dev/my-project
cp ~/dot_files_nixos/devenv-example/devenv.nix .
echo 'use devenv' > .envrc
direnv allow
```

Available versions in `devenv.nix`:
- Ruby: `pkgs.ruby_3_2`, `pkgs.ruby_3_3`, `pkgs.ruby_3_4`
- Node: `pkgs.nodejs_20`, `pkgs.nodejs_22`

Direnv activates the environment automatically when you enter the directory.

```bash
devenv up           # start PostgreSQL + Redis in background
bundle exec rails s
# or
overmind start      # start everything via Procfile.dev
```

## Neovim

LazyVim config (`~/.nv-ide`) is kept unchanged via symlink.
Mason is disabled: LSPs/formatters are installed via nix and available in PATH.

| Lua LSP | `lua-language-server` |

| HTML/CSS/JSON/ESLint | `vscode-langservers-extracted` |

## Hyprland

Hyprland is available as an alternative session to Sway, selectable from dms-greeter.
Config in `home/programs/hyprland.nix` (laptop) and `hyprland-desktop.nix` (desktop).

Keybindings are identical to Sway. The only differences:
- Layout: `dwindle` (instead of i3-style tiling) with resize submap (`SUPER+R`)
- Screenshot: `grim -g "$(slurp)"` instead of `grimshot`
- dms still starts via systemd and manages bar, spotlight, audio, brightness

### Hyprland monitors

Update the `monitor` section in `hyprland.nix` based on your hardware.
Use `hyprctl monitors` to identify connector names.

```bash
hyprctl monitors        # list connected monitors with names and resolutions
hyprctl activewindow    # active window info (useful for window rules)
hyprctl reload          # reload config without restarting (SUPER+SHIFT+C)
```

## DankMaterialShell (dms)

DMS handles: spotlight (`Mod+Space`), clipboard (`Mod+v`), processlist (`Mod+m`),
settings (`Mod+,`), volume (`XF86Audio*`), brightness (`XF86Brightness*`).

Enabled as a NixOS module in `configuration.nix`. Starts automatically via systemd.
`dms-greeter` is active on both hosts as the display manager.

## Notes

### Monitors
Update the `output` section in the sway program files based on your hardware.
Use `wdisplays` to identify output names (`eDP-1`, `DP-1`, etc.).

### Display manager
`dms-greeter` active on both hosts. Sway, Hyprland and GNOME (desktop only) are all selectable as sessions at login.

### DBeaver data
DBeaver data is not synced. Copy from old PC:
```bash
scp -r old-pc:~/.local/share/DBeaverData ~/.local/share/DBeaverData
```

### gcloud CLI
Configure account after first launch:
```bash
gcloud auth login
gcloud config set project <project-id>
```

### Atuin: sync shell history
```bash
atuin login
atuin sync
```

### GitHub CLI (gh): authenticate
```bash
gh auth login --web
```
Don't pass `--git-protocol ssh`: it's already set declaratively in `home/programs/git.nix` (`programs.gh.settings.git_protocol`), which home-manager symlinks read-only into `~/.config/gh/config.yml`. Passing that flag makes `gh` try to rewrite the file and fail with `read-only file system` — harmless noise, login still succeeds (the token is saved in the writable `hosts.yml`), but skip the flag to avoid the error.
